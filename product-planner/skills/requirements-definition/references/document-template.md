# 要件定義ドキュメント テンプレート

Phase 4 でこの構造に沿って `docs/requirements/<slug>.md` を生成する。
各セクションは、対応するフェーズでユーザーが確定した内容のみを記載する。未確定の推測は書かない。

```markdown
# {プロダクト名} 要件定義書

## 概要

- **解決する課題**: {Phase 1}
- **対象ユーザー**: {Phase 1}
- **参考にした類似プロダクト・事例**: {Phase 1、なければ「特になし」}

## ゴール

{このプロダクトが達成したい状態を1〜3文で}

## 機能要件

| ID  | 機能 | ユーザーストーリー                   | 優先度 |
| --- | ---- | ------------------------------------ | ------ |
| F1  | ...  | 〜として、〜したい。なぜなら〜だから | MUST   |

## 非機能要件

{Phase 3 で選択されたカテゴリのみ、以下の見出しで記載。選択されなかったカテゴリの見出しは含めない}

### パフォーマンス

- ...

### セキュリティ

- ...

### 可用性

- ...

### スケーラビリティ

- ...

### 保守性・拡張性

- ...

### コスト

- ...

### 運用・監視

- ...

## スコープ外

{今回は対象外とすることが確認された事項。なければ「未確定（今後の議論で明確化）」}

## 次のステップ

この要件定義書をもとに実装タスクへ分解する場合は、`task-planner:task-breakdown` スキルを起動する。
```
