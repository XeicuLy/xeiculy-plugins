# 読書ログ 要件定義書

## 概要

- **解決する課題**: 読んだ本の感想や気づきをすぐ忘れてしまい、後から振り返れない
- **対象ユーザー**: 月に2〜5冊程度読む個人の読書家。スマートフォンで隙間時間に記録したい層
- **参考にした類似プロダクト・事例**: Goodreads、読書メーター（ただし本アプリはSNS機能を持たない個人用途に特化）

## ゴール

読んだ本とその感想を1分以内に記録でき、過去の記録を本棚形式で振り返れる状態を実現する。

## 機能要件

| ID  | 機能         | ユーザーストーリー                                                           | 優先度 |
| --- | ------------ | ---------------------------------------------------------------------------- | ------ |
| F1  | 本の登録     | 読者として、タイトル・著者・読了日を登録したい。なぜなら記録を残したいから   | MUST   |
| F2  | 感想メモ     | 読者として、感想を自由記述で残したい。なぜなら後で振り返りたいから           | MUST   |
| F3  | 本棚一覧表示 | 読者として、登録した本を一覧で見たい。なぜなら読書履歴を振り返りたいから     | MUST   |
| F4  | タグ付け     | 読者として、本にジャンルタグを付けたい。なぜなら後で絞り込みたいから         | SHOULD |
| F5  | 読書統計     | 読者として、月ごとの読了冊数をグラフで見たい。なぜなら習慣を可視化したいから | COULD  |

## 非機能要件

### パフォーマンス

- 本棚一覧（最大500冊）が1秒以内に表示されること

### セキュリティ

- 個人の読書記録のみを扱うため、機密情報や決済情報は扱わない
- 端末ロック以外の追加認証は不要とする

### 可用性

- 個人利用ツールのため、稼働率の厳密な目標値は設けない
- オフライン時も直近の記録は閲覧・追記できること

## スコープ外

- 他ユーザーとの共有・SNS機能
- 電子書籍の閲覧機能そのもの
